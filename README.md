# EVERYDAY POS — APK для терминала Sunmi V3

APK-обёртка вокруг **уже готового сайта** `eposdemo.netlify.app`.
Делает две вещи:

1. Открывает сайт на весь экран как приложение (без адресной строки браузера).
2. Даёт сайту доступ к **встроенному термопринтеру Sunmi** — чтобы печатать чек по кнопке.

Сам сайт не тронут. Это тонкая «рамка» + мост к принтеру.

---

## Как это работает (коротко)

- Внутри приложения открыт наш сайт (WebView).
- Когда на сайте жмут «Печать чека», сайт зовёт `window.AndroidPrinter.printReceipt(...)`.
- Этот вызов ловит [`PrinterBridge.java`](app/src/main/java/com/everyday/pos/PrinterBridge.java) и печатает чек на принтере Sunmi через их SDK.
- В обычном браузере (на компе) моста нет — там сайт печатает как раньше (`window.print`). Ничего не сломалось.

Если поменяется домен сайта — поправь `SITE_URL` в
[`MainActivity.java`](app/src/main/java/com/everyday/pos/MainActivity.java).

## Как собрать APK (без установки программ) — через GitHub

Ты собираешь APK в облаке GitHub бесплатно. Android Studio ставить НЕ нужно.

### Шаг 1. Создай репозиторий
1. Зайди на https://github.com , нажми **New repository**.
2. Имя любое, например `everyday-pos`. Можно **Private**. Создай.

### Шаг 2. Залей эту папку в репозиторий
Самый простой путь без командной строки:
1. На странице пустого репозитория нажми **uploading an existing file**.
2. Перетащи в окно **всё содержимое папки `EPAY-APK`** (не саму папку, а то, что внутри: `app`, `gradle`, `gradlew`, `build.gradle` и т.д.).
   > Важно: папка `.github` тоже должна залиться — в ней рецепт сборки. Если браузер её прячет, см. вариант с командной строкой ниже.
3. Внизу нажми **Commit changes**.

### Шаг 3. Дождись сборки
1. Открой вкладку **Actions** в репозитории.
2. Увидишь запущённую задачу **Build APK** (крутится жёлтый кружок). Идёт ~3-5 минут.
3. Когда станет зелёная галочка — открой этот запуск.

### Шаг 4. Скачай APK
1. Внизу страницы запуска, в разделе **Artifacts**, будет файл **EverydayPOS-apk**.
2. Скачай его — внутри `app-debug.apk`.

### Шаг 5. Установи на терминал Sunmi
1. Перекинь `app-debug.apk` на терминал (USB-флешка / Telegram себе / скачать прямо в браузере терминала).
2. Открой файл на терминале, разреши «установку из этого источника», установи.
3. Запусти **Everyday POS** — откроется наш сайт на весь экран.
4. Проведи оплату → «Печать чека» → должно напечатать на встроенном принтере.

## Если хочешь заливать через командную строку (надёжнее для папки .github)

Установи Git (https://git-scm.com), потом в папке `EPAY-APK`:

```bash
git init
git add .
git commit -m "Everyday POS APK"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЛОГИН/everyday-pos.git
git push -u origin main
```

Дальше — вкладка **Actions**, шаги 3-5 выше.

## Частые вопросы

**«Принтер Sunmi не подключён»** при печати — приложение не нашло службу принтера.
Убедись, что APK стоит именно на терминале Sunmi (на обычном телефоне принтера нет),
и что это модель со встроенным принтером (V3 — да).

**Хочу подписанный релиз (для магазина/раздачи)** — сейчас собирается debug-APK, его
достаточно для установки на свой терминал. Для release-подписи нужен keystore — скажи, добавлю.

**Обновил сайт — надо пересобирать APK?** Нет. Сайт грузится с Netlify вживую.
APK пересобирается только если менялся `SITE_URL` или код принтера.
